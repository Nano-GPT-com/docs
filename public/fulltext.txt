# NanoGPT API Documentation

## Introduction

The NanoGPT API allows you to generate text, images and video using any AI model available. Our implementation for text generation generally matches the OpenAI standards.

## Endpoints

### Data API (GET/POST /v1/data)

Unified discovery and dispatch layer for NanoGPT data tools. Use `GET /api/v1/data` to list available tools, `POST /api/v1/data/{path}` for path-style dispatch, or `POST /api/v1/data` with an `endpoint`/`id`/`type` field for body-style dispatch.

The Data API wraps existing direct endpoints for web search, URL scraping, Firecrawl, Google Maps, Reddit, LinkedIn, Instagram, TikTok, X/Twitter, and Hunter enrichment. It preserves underlying response bodies, validation, billing, rate limits, async behavior, provider options, authentication, and x402 behavior. Direct endpoints remain available.

**Examples:**
```
GET https://nano-gpt.com/api/v1/data
Authorization: Bearer YOUR_API_KEY
```

```
POST https://nano-gpt.com/api/v1/data/web/search
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

{"query":"latest AI model releases","depth":"standard"}
```

```
POST https://nano-gpt.com/api/v1/data
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

{"endpoint":"web.search","input":{"query":"latest AI model releases","depth":"standard"}}
```

Dispatched responses include `x-nanogpt-data-endpoint` and `x-nanogpt-direct-endpoint`.

### Models (GET /v1/models)

OpenAI-compatible endpoint for retrieving available models with optional detailed information including pricing.

**Authentication:** Bearer token (optional, enables user-specific pricing in detailed mode)

**Query Parameters:**
- `detailed` (boolean, default: false): Returns detailed model information including pricing
- `sort` (string): Values: `favorites`, `mostused`. Reorders the returned model list. Sorting only affects models that would already be visible to the caller. It does not add or hide models.

**Endpoint Behavior:**
- `GET /api/v1/models` supports `sort=favorites` and `sort=mostused`.
- `sort=favorites`: Orders models by the authenticated account's recent model usage.
- `sort=mostused`: Orders models by global completed model usage.
- Unused or unranked models remain in the response after ranked models, in the normal catalog order.
- `sort=favorites` requires a valid API key to personalize the order. If no valid API key is provided, it falls back to normal ordering.
- `sort=mostused` does not require authentication.
- Invalid `sort` values are ignored.
- Can be combined with `detailed=true`.

**Favorites-Sorted Request:**
```
GET https://nano-gpt.com/api/v1/models?sort=favorites
Authorization: Bearer YOUR_API_KEY
```

**Most-Used Request:**
```
GET https://nano-gpt.com/api/v1/models?sort=mostused
```

**Favorites-Sorted Detailed Request:**
```
GET https://nano-gpt.com/api/v1/models?detailed=true&sort=favorites
Authorization: Bearer YOUR_API_KEY
```

**Most-Used Detailed Request:**
```
GET https://nano-gpt.com/api/v1/models?detailed=true&sort=mostused
```

**Endpoint Variants:**
- `GET /api/v1/models` (canonical): Returns all visible text models. If your account has a subscription and you have not enabled "Also show paid models", this list is automatically restricted to subscription-included models.
- `GET /api/subscription/v1/models` (subscription-only): Always returns only models included in the NanoGPT subscription; ignores the "Also show paid models" preference. Supports `?detailed=true`.
- `GET /api/paid/v1/models` (paid/extras): Returns only models not included in the subscription. Supports `?detailed=true`.

**Distillation Policy:**
Text model records and provider rows may include `distillationPolicy`, an API-only object with `status`, `label`, `basis`, `sourceUrl`, and `note`. It indicates whether model or provider-route outputs may be used for output-based model training or distillation under NanoGPT's recorded model-license and provider-terms rules.

The explore text model endpoints support `distillation=allowed`:
```
GET https://nano-gpt.com/api/explore/text-models?distillation=allowed
GET https://nano-gpt.com/api/explore/search?type=text&q=qwen&distillation=allowed
```

The filter applies only to text models. Responses include `meta.distillation: "allowed"` when the filter is used and `meta.distillation: "all"` otherwise. Provider-specific restrictions override model-level allowances, so inspect each selected provider route's `distillationPolicy.sourceUrl` and `distillationPolicy.note` before relying on it.

**Example Basic Request:**
```
GET https://nano-gpt.com/api/v1/models
Authorization: Bearer YOUR_API_KEY
```

**Example Basic Response:**
```json
{
  "object": "list",

    {
      "id": "openai/gpt-5.2",
      "object": "model",
      "created": 1677649963,
      "owned_by": "openai"
    }
  ]
}
```


**Subscription-Only List:**

### Batch API (POST /v1/batches)

OpenAI-compatible asynchronous batch jobs for high-volume chat completion workloads where latency is not important.

**Base URL:** `https://api.nano-gpt.com/api/v1`

Use `api.nano-gpt.com` for file upload, batch creation, polling, cancellation, and output downloads. Do not use `nano-gpt.com` for batch uploads, because larger multipart uploads can be rejected before they reach the Batch API.

**Authentication:** Bearer token

**Supported endpoints:**
- Files: `POST /files`, `GET /files/{file_id}`, `GET /files/{file_id}/content`
- Batches: `POST /batches`, `GET /batches/{batch_id}`, `GET /batches`, `POST /batches/{batch_id}/cancel`

**Supported JSONL row endpoint:** `/v1/chat/completions`

Batch rows support the listed GPT/OpenAI, Claude, and Gemini chat models. GPT, Claude, and Gemini batch jobs support text prompts and OpenAI-compatible `image_url` image inputs in user messages. Image URLs may be `http://`, `https://`, or base64 `data:image/...;base64,...` URLs for PNG, JPEG, GIF, and WebP.

**Request rules:**
- Upload files with `purpose=batch`.
- Each JSONL row must include a unique non-empty `custom_id`.
- Each row must use `method: "POST"` and `url: "/v1/chat/completions"`.
- Each row must include `body.messages`, `body.model`, and `body.max_tokens` or `body.max_completion_tokens`.
- All rows in one batch must use the same model and model family. Do not mix GPT, Claude, and Gemini rows.
- Streaming, tools/functions, `response_format`, audio, and non-image multimodal request bodies are not supported.

**Example JSONL row:**
```jsonl
{"custom_id":"request-1","method":"POST","url":"/v1/chat/completions","body":{"model":"gpt-4.1-mini","messages":[{"role":"user","content":"Summarize this in one sentence."}],"max_tokens":64}}
```

**Upload:**
```bash
curl https://api.nano-gpt.com/api/v1/files \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -F purpose=batch \
  -F file=@batch.jsonl
```

**Create batch:**
```bash
curl https://api.nano-gpt.com/api/v1/batches \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"input_file_id":"file_abc123","endpoint":"/v1/chat/completions","completion_window":"24h"}'
```

**Statuses:** `validating`, `in_progress`, `finalizing`, `completed`, `failed`, `expired`, `cancelling`, `cancelled`

Batch jobs use NanoGPT account balance. Subscription included-token logic is not applied. NanoGPT checks account balance against a conservative maximum liability estimate at batch creation, then bills completed usage once after the batch reaches a terminal status.

### Chat Completion (POST /v1/chat/completions)

OpenAI-compatible endpoint for chat completions.

**Authentication:** Bearer token

**Cache-capable provider routing:** Set top-level `caching: true`, or append `:caching` / `:cache` / `:cached` to the model, when the request must route to any available provider that supports prompt/input caching. If no usable cache-capable provider exists for the model, the request fails instead of using a non-caching provider. `caching: true` defaults to sticky provider routing for later matching requests from the same API key or session; set `stickyprovider: false` to opt out. `stickyProvider` is also accepted as a camelCase alias. This does not add prompt-cache annotations, configure cache TTLs, or force a cache write; use `prompt_caching` / `promptCaching` for explicit provider-specific cache controls.

**Provider routing object:** The body `provider` field accepts either a provider ID string or an object with `order`, `only`, `ignore`, `sort`, `quantizations`, `min_quantization`, `max_price`, `allow_fallbacks`, and `require_parameters`. `order` is a soft preference, `only` is a hard pin, `max_price` uses USD per 1 million tokens, and quantization filters are hard routing constraints.

**Example Request:**
```json
POST https://nano-gpt.com/api/v1/chat/completions
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

{
  "model": "openai/gpt-5.2",
  "messages": [
    {
      "role": "system",
      "content": "You are a helpful assistant."
    },
    {
      "role": "user",
      "content": "Please explain the concept of artificial intelligence."
    }
  ],
  "stream": true,
  "temperature": 0.7,
  "max_tokens": 4000,
  "top_p": 1,
  "frequency_penalty": 0,
  "presence_penalty": 0
}
```

**Example Response:**
```json
{
  "id": "chatcmpl-123",
  "object": "chat.completion",
  "created": 1677652288,
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Artificial intelligence (AI) is..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 9,
    "completion_tokens": 12,
    "total_tokens": 21
  }
}
```

### Usage (GET /v1/usage)

Retrieve aggregate spend, refund, request, and token usage for the authenticated API key.

**Authentication:** Bearer token

**Default behavior:** If no date range is provided, returns the current API key's last 30 UTC days grouped by both day and model.

**Query Parameters:**
- `from` (string, YYYY-MM-DD): UTC start date. Must be provided with `to`.
- `to` (string, YYYY-MM-DD): UTC end date, inclusive. Must be on or after `from`, cannot be in the future, and ranges are capped at 366 days.
- `group_by` (string, default: `day,model`): Values: `day`, `model`, `day,model`. `model,day` is accepted as an alias.
- `scope` (string, default: `current_key`): `current_key` and `api_key` both return the authenticated API key's usage.
- `api_key_id` (integer): Optional current API key ID. Requests for another API key are rejected.

The endpoint returns aggregates only. It does not return raw transaction rows, prompts, responses, provider routing details, or account-wide usage across multiple API keys.

**Example Request:**
```
GET https://nano-gpt.com/api/v1/usage?from=2026-05-01&to=2026-05-31&group_by=model
Authorization: Bearer YOUR_API_KEY
```

**Example Response:**
```json
{
  "object": "usage",
  "scope": "current_key",
  "apiKey": { "id": 123 },
  "from": "2026-05-01",
  "to": "2026-05-31",
  "timezone": "UTC",
  "groupBy": "model",
  "asOf": "2026-05-31T12:00:00.000Z",
  "source": {
    "rollupDays": ["2026-05-01"],
    "liveDays": ["2026-05-31"],
    "missingRollupDays": []
  },
  "totals": {
    "requests": 1284,
    "costUsd": 42.18,
    "refundedUsd": 1.25,
    "netCostUsd": 40.93,
    "inputTokens": 1234567,
    "outputTokens": 456789,
    "reasoningTokens": 12000,
    "totalTokens": 1691356
  },
  "byModel": [
    {
      "model": "GPT-4.1 mini",
      "requests": 812,
      "costUsd": 18.92,
      "refundedUsd": 0.5,
      "netCostUsd": 18.42,
      "inputTokens": 900000,
      "outputTokens": 220000,
      "reasoningTokens": 0,
      "totalTokens": 1120000
    }
  ]
}
```

`totalTokens` is `inputTokens + outputTokens`; reasoning tokens are reported separately. `netCostUsd` is `max(0, costUsd - refundedUsd)` per aggregate bucket, so row sums can differ slightly from `totals.netCostUsd` when refunds cross buckets.

## Integrations

### Cline

Cline is a powerful command-line interface for interacting with various AI models, including those available through NanoGPT.

**Setup Instructions:**

1. Get your API key from [nano-gpt.com/api](https://nano-gpt.com/api)
2. In Cline settings, add a new Custom Model with these details:
   - API Provider: OpenAI Compatible
   - Base URL: https://nano-gpt.com/api/v1/
   - API Key: Your key from step 1
   - Model ID: openai/gpt-5.2

Available models include openai/gpt-5.2, anthropic/claude-opus-4.5, google/gemini-3-flash-preview, and many more.

### Cursor

Cursor is an AI-powered code editor that enhances coding productivity.

**Setup Instructions:**

1. Get your API key from [nano-gpt.com/api](https://nano-gpt.com/api)
2. In Cursor settings:
   - Override OpenAI Base URL with: https://nano-gpt.com/api/v1/
   - Set the API Key to your NanoGPT API key
   - Click "+ Add Model" to add any of our models you want to use

### SillyTavern

SillyTavern is a popular web-based chat interface designed for roleplaying and character-based interactions.

**Setup Instructions:**

1. Launch SillyTavern and navigate to the "API Connections" tab.
2. Configure NanoGPT:
   - Select "OpenAI" as the API type (NanoGPT uses an OpenAI-compatible API)
   - Enter your NanoGPT API key
   - Set the API URL to: `https://nano-gpt.com/api`
   - Set the Model to: `openai/gpt-5.2` (or another model of your choice)

If you choose the Chat Completions endpoint in SillyTavern, you will see NanoGPT automatically appear as an option.

### OpenWebUI

OpenWebUI is an open-source web interface for interacting with AI models.

**Setup Instructions:**

1. Get your API key from [nano-gpt.com/api](https://nano-gpt.com/api)
2. In OpenWebUI, go to Settings and find the "Add Connection" option
3. Enter the following details:
   - URL: `https://nano-gpt.com/api/v1`
   - API Key: Your key from step 1
   - Prefix ID: (Optional) Leave empty or add a custom prefix
   - Model IDs: Leave empty to include all available models

### TypingMind

TypingMind is a powerful alternative web UI for ChatGPT with advanced features.

**Setup Instructions:**

1. Get your API key from your [API page](https://nano-gpt.com/api)
2. In TypingMind, go to add custom model
3. Select "Create Manually"
4. Enter the following details:
   - Name: NanoGPT
   - API Type: OpenAI Compatible API
   - Icon URL (Optional): https://nano-gpt.com/logo.png
   - Endpoint: `https://nano-gpt.com/api/v1/chat/completions`
   - Model ID: openai/gpt-5.2 (or any other model)
   - Authorization: Bearer YOUR_API_KEY
5. Click "Test" to verify the connection, then "Add Model"

## Miscellaneous Information

### Inline Moderation

Inline moderation lets authenticated API-key callers send `moderation: true` or `moderation: <model-id>` to run a paid input safety preflight before selected text, image, and video generation requests are dispatched to providers. It is input-only and does not moderate generated outputs.

Supported authenticated routes include `POST /api/v1/chat/completions`, `POST /api/v1/responses`, `POST /api/v1/completions`, `POST /api/v1/messages`, `POST /api/v1/images/generations`, `POST /api/v1/images/edits`, image generation compatibility routes that use the canonical image handler, and video generation requests that use the canonical video handler.

Use `GET /api/v1/moderation-models` to list moderation model IDs. `moderation-model` only selects a model when the `moderation` header is also present; conflicting model IDs return `conflicting_moderation_model`.

If input passes, NanoGPT charges the moderation check and continues with normal generation billing. If input is flagged, NanoGPT charges the moderation check, skips generation, does not charge generation, and returns `400 content_policy_violation`. Accountless x402/exact-payment requests are not supported and return `inline_moderation_requires_api_key`.

Image inputs must be inspectable HTTP(S) URLs or `data:image/...` URLs. Unsupported image/file references fail closed with `unsupported_moderation_input`.

### Client Libraries

#### JavaScript - NanoGPTJS

[NanoGPTJS](https://github.com/kilkelly/nanogptjs) is a Node.js library to interact with NanoGPT's API. The NanoGPT service enables pay-per-prompt interaction with chat and image generation models. You will need a prefilled NanoGPT wallet and API key to use this library effectively.

**Installation:**
```bash
npm install nanogptjs
```

**Basic Usage:**
```javascript
const NanoGPT = require('nanogptjs');
const nanogpt = new NanoGPT('your-api-key');

async function chatExample() {
  const response = await nanogpt.chat({
    model: 'openai/gpt-5.2',
    messages: [
      { role: 'user', content: 'Hello!' }
    ]
  });
  console.log(response);
}
```

#### TypeScript - NanoGPT-client

[NanoGPT-client](https://github.com/aspic/nanogpt-client) is an unofficial implementation of the NanoGPT API. This library aims to provide a TypeScript client for both browser and node environments. It's built on the inferred OpenAPI spec.

**Installation:**
```bash
npm install nanogpt-client
```

**Basic Usage:**
```typescript
import { NanoGPTClient } from 'nanogpt-client';

const client = new NanoGPTClient({
  apiKey: 'your-api-key'
});

async function main() {
  const response = await client.chatCompletions.create({
    model: 'openai/gpt-5.2',
    messages: [
      { role: 'user', content: 'Hello!' }
    ]
  });
  console.log(response.choices[0].message.content);
}
```

### Rate Limits

The NanoGPT API currently does not impose any rate limits on API requests. You can make as many requests as needed without worrying about hitting artificial limits.

However, we recommend implementing reasonable request patterns in your applications to ensure optimal performance and fairness for all users.
