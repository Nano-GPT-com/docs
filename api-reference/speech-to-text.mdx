## Overview

Use the OpenAI-compatible endpoint for new speech-to-text integrations:

```http
POST /api/v1/audio/transcriptions
```

It accepts one audio source as a multipart file, a remote URL, a base64 data URI, or a structured `input_audio` JSON object. The `model` field is optional and defaults to `Whisper-Large-V3`.

Authenticate with your NanoGPT API key:

```http
Authorization: Bearer $NANOGPT_API_KEY
```

## Discover models

Retrieve the current public models that support synchronous transcription:

```bash
curl "https://nano-gpt.com/api/v1/models?output_modalities=transcription" \
  -H "Authorization: Bearer $NANOGPT_API_KEY"
```

Use this discovery request instead of maintaining a static model list or using `?detailed=true`. Returned entries can describe pricing, supported formats and languages, remote URL support, model-specific file limits, timestamps, diarization, and other speech-to-text capabilities.

The selector excludes voice-cloning tools and models that require asynchronous polling.

## Input limits

> **Input limits:** The complete direct multipart or JSON request body is limited to 4 MiB, including multipart boundaries, form fields, the JSON envelope, and base64 expansion. Base64 JSON therefore has a practical decoded-audio ceiling of about 3 MiB. For larger media, provide a remotely reachable URL. Remote media limits vary by model; check `supported_parameters.max_file_size_mb` in `GET /api/v1/models?output_modalities=transcription`.

## Input methods

Each request must contain exactly one audio source.

### Multipart file

Upload a small audio file in the `file` field. The complete multipart body, not just the file, must fit within 4 MiB.

```bash
curl https://nano-gpt.com/api/v1/audio/transcriptions \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -F "file=@speech.wav" \
  -F "model=Whisper-Large-V3" \
  -F "response_format=verbose_json" \
  -F "timestamp_granularities[]=word"
```

See the [multipart fields and examples](/api-reference/endpoint/audio-transcriptions#multipart-fields) for optional parameters and aliases.

### Remote URL

Use a remotely reachable URL for media that would exceed the direct request limit. The URL can be supplied as `file_url`, `audio_url`, or `audioUrl` in JSON or multipart requests.

```bash
curl https://nano-gpt.com/api/v1/audio/transcriptions \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "file_url": "https://example.com/speech.mp3",
    "model": "Whisper-Large-V3",
    "language": "en"
  }'
```

See the [JSON URL example](/api-reference/endpoint/audio-transcriptions#json-url-example).

### Structured base64 audio

Send raw base64 audio bytes in `input_audio.data` and identify the format in `input_audio.format`. The camel-case alias `inputAudio` is also accepted.

```bash
curl https://nano-gpt.com/api/v1/audio/transcriptions \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Whisper-Large-V3",
    "input_audio": {
      "data": "BASE64_AUDIO_BYTES",
      "format": "wav"
    }
  }'
```

Supported structured formats are `aac`, `flac`, `m4a`, `mp3`, `mp4`, `mpeg`, `mpga`, `ogg`, `opus`, `wav`, and `webm`. See the [structured base64 example](/api-reference/endpoint/audio-transcriptions#structured-base64-example).

### Base64 data URI

The URL aliases also accept a supported base64 audio data URI:

```json
{
  "model": "Whisper-Large-V3",
  "audio_url": "data:audio/wav;base64,BASE64_AUDIO_BYTES"
}
```

Base64 expansion and the JSON envelope count toward the 4 MiB request-body limit.

## Responses

The default `json` response always includes `text`. It can also include `language`, measured media `duration`, and sanitized `usage` when those values are available.

```json
{
  "text": "Transcribed text",
  "language": "en",
  "duration": 42.1,
  "usage": {
    "type": "duration",
    "seconds": 42.1,
    "cost": 0.001,
    "currency": "USD"
  }
}
```

`duration` is measured media duration in seconds, not a rounded or billable duration. It may be omitted when it cannot be measured. Successful responses can also include `x-generation-id` and `x-request-id` headers.

Use `response_format` to request `json`, `verbose_json`, `text`, `srt`, or `vtt`, where supported by the selected model. See [response fields and formats](/api-reference/endpoint/audio-transcriptions#responses) for details.

## Legacy endpoint

`POST /api/transcribe` remains available for existing integrations and has its own request and response fields. See the [legacy transcription endpoint](/api-reference/endpoint/transcribe).

`actualDuration` is a legacy `/api/transcribe` field. The OpenAI-compatible `/api/v1/audio/transcriptions` multipart endpoint rejects it.
