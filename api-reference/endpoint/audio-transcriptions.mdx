Transcribe audio with an OpenAI-compatible endpoint. Supply exactly one audio source as a multipart file, a remote URL, a base64 data URI, or a structured `input_audio` object.

Authenticate with your NanoGPT API key:

```http
Authorization: Bearer $NANOGPT_API_KEY
```

The `model` field is optional and defaults to `Whisper-Large-V3`. To retrieve the current list of public synchronous transcription models and their capabilities, use:

```http
GET /api/v1/models?output_modalities=transcription
```

## Request size limit

The complete direct multipart or JSON request body must not exceed **4 MiB (4,194,304 bytes)**. Multipart boundaries, form fields, the JSON envelope, and base64 expansion all count toward this limit.

Because base64 adds roughly 33% overhead, a base64 JSON request can usually contain only about **3 MiB of decoded audio**, with the exact ceiling depending on the rest of the JSON body. For larger media, provide a remotely reachable URL with `file_url` or `audio_url`. Remote media is still subject to the selected model's `max_file_size_mb` limit.

## Multipart fields

Send `multipart/form-data` with exactly one audio source.

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `file` | file | One source required | Audio uploaded directly. The complete request must fit within 4 MiB. |
| `file_url` | string | One source required | Remotely reachable audio URL. Aliases: `audio_url`, `audioUrl`. |
| `model` | string | No | Speech-to-text model ID. Defaults to `Whisper-Large-V3`. |
| `language` | string | No | Input language hint supported by the selected model. |
| `prompt` | string | No | Optional transcription prompt supported by the selected model. |
| `task` | string | No | `transcribe` or `translate`. Support varies by model. |
| `response_format` | string | No | `json`, `verbose_json`, `text`, `srt`, or `vtt`. Alias: `responseFormat`. Support varies by model. |
| `temperature` | number | No | Sampling temperature supported by the selected model. |
| `timestamp_granularities[]` | string | No | Repeat with `word` and/or `segment`. Also accepted as `timestamp_granularities` or `timestampGranularities`. Support varies by model. |

Unknown multipart fields are rejected. `actualDuration` is a legacy `/api/transcribe` field and is not accepted by this OpenAI-compatible endpoint.

### Multipart file example

```bash
curl https://nano-gpt.com/api/v1/audio/transcriptions \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -F "file=@speech.wav" \
  -F "model=Whisper-Large-V3" \
  -F "response_format=verbose_json" \
  -F "timestamp_granularities[]=word"
```

### Multipart URL example

```bash
curl https://nano-gpt.com/api/v1/audio/transcriptions \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -F "file_url=https://example.com/speech.mp3" \
  -F "model=Whisper-Large-V3"
```

## JSON requests

Send `application/json` with exactly one of these source forms:

- A remote URL or base64 data URI in `file_url`, `audio_url`, or `audioUrl`.
- A structured base64 object in `input_audio` or `inputAudio`.

Structured `input_audio.format` accepts: `aac`, `flac`, `m4a`, `mp3`, `mp4`, `mpeg`, `mpga`, `ogg`, `opus`, `wav`, and `webm`.

### JSON URL example

```bash
curl https://nano-gpt.com/api/v1/audio/transcriptions \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "file_url": "https://example.com/speech.mp3",
    "model": "Whisper-Large-V3",
    "language": "en"
  }'
```

### Structured base64 example

```bash
curl https://nano-gpt.com/api/v1/audio/transcriptions \
  -H "Authorization: Bearer $NANOGPT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Whisper-Large-V3",
    "input_audio": {
      "data": "BASE64_AUDIO_BYTES",
      "format": "wav"
    }
  }'
```

A data URI can be supplied directly through a URL alias:

```json
{
  "model": "Whisper-Large-V3",
  "audio_url": "data:audio/wav;base64,BASE64_AUDIO_BYTES"
}
```

Do not send more than one source type in the same request.

## Responses

For `json`, the endpoint returns a JSON object. Fields that are unavailable are omitted:

```json
{
  "text": "Transcribed text",
  "language": "en",
  "duration": 42.1,
  "usage": {
    "type": "duration",
    "seconds": 42.1,
    "cost": 0.001,
    "currency": "USD"
  }
}
```

- `text` is always present in a successful JSON response.
- `language` is included when available.
- `duration` is measured media duration in seconds. It is not a rounded or billable duration and may be omitted when the duration cannot be measured.
- `usage` is included when usage or billing information is available. `cost` is the NanoGPT customer charge and `currency` is the actual billing ticker, such as `USD` or `XNO`.
- Duration-based usage has `type: "duration"` and `seconds`. When token counts are available, usage has `type: "tokens"` and can include `input_tokens`, `output_tokens`, and `total_tokens`.
- `verbose_json` can additionally include `task`, `language_probability`, `segments`, and `words`. In verbose output, unknown `language` or `duration` may be `null`.
- `text`, `srt`, and `vtt` response formats return `text/plain` rather than JSON.

Successful responses can include these headers:

| Header | Description |
| --- | --- |
| `x-generation-id` | Generation or upstream request identifier, when available. Exposed to browser clients through CORS. |
| `x-request-id` | NanoGPT or upstream request identifier, when available. |

## Errors

Errors use an OpenAI-style envelope:

```json
{
  "error": {
    "message": "Request body too large. Use 'file_url' or 'audio_url' for larger media.",
    "type": "invalid_request_error",
    "param": "body",
    "code": "request_body_too_large"
  }
}
```

| Status | Meaning |
| --- | --- |
| `400` | Missing, conflicting, invalid, or unsupported request fields. |
| `401` | Missing or invalid API credentials. |
| `402` | Payment is required. |
| `413` | The complete direct request body exceeds 4 MiB. Use a remote URL. |
| `415` | Unsupported content type. Use `multipart/form-data` or `application/json`. |
| `429` | Rate limit exceeded. |
