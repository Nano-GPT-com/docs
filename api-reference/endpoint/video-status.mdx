## Overview

Video generation is asynchronous. After you submit a request to `POST /generate-video`, poll this endpoint with the returned `runId` and the `modelSlug` you used. When the run completes, the video URL is available at `data.output.video.url`.

### Query parameters

- `runId` (string, required): Run identifier returned from the submit request
- `modelSlug` (string, required): Model used for the run (for example, `veo2-video`)

### Notes

- Poll every 5 seconds, up to ~10 minutes
- Status values: `IN_QUEUE`, `IN_PROGRESS`, `COMPLETED`, `FAILED`, `CANCELLED`
- Polling this endpoint is not rate limited; submission endpoints are

### Examples

<CodeGroup>

```bash cURL
# Single status check
curl -s "https://nano-gpt.com/api/generate-video/status?runId=YOUR_RUN_ID&modelSlug=veo2-video" \
  -H "x-api-key: YOUR_API_KEY" | jq .
```

```javascript JavaScript
const BASE = 'https://nano-gpt.com/api';

export async function getVideoStatus(runId, modelSlug, apiKey) {
  const res = await fetch(`${BASE}/generate-video/status?runId=${runId}&modelSlug=${modelSlug}`, {
    headers: { 'x-api-key': apiKey }
  });
  if (!res.ok) throw new Error(`Status check failed: ${res.status}`);
  return res.json();
}
```
